# Ручные тест-кейсы Enduro Park Manager (QA-Stand)

**Версия:** 1.2  
**Дата:** 13.07.2026  
**Изменения от v1.1:** убраны guest и связанные кейсы; главная публична; VIN редактируем; новые testid.

---

## 1. Аутентификация

### TC-AUTH-01: Вход под admin
**Предусловия:** seed выполнен (`admin` / `admin123`).  
**Шаги:** `/login` → ввести admin / admin123 → «Войти».  
**Ожидание:** редирект на `/`; `user-username`=admin; `user-role`=admin; `logout-btn`; httpOnly cookie.

### TC-AUTH-02: Вход под mechanic
**Данные:** mechanic / admin123.  
**Ожидание:** как TC-AUTH-01, role=mechanic.

### ~~TC-AUTH-03~~ — **УДАЛЁН** (роль guest снята)

### TC-AUTH-04: Неверный пароль
**Данные:** admin / wrongpassword.  
**Ожидание:** остаёмся на `/login`; `login-error-message`; нет cookie.

### TC-AUTH-05: Несуществующий пользователь
**Данные:** unknown / anypassword.  
**Ожидание:** как TC-AUTH-04.

### TC-AUTH-06: Выход
**Предусловия:** залогинен любой пользователь.  
**Шаги:** `logout-btn`.  
**Ожидание:** cookie очищена; остаёмся на `/`; таблица видна; `header-login-btn` вместо блока пользователя.

### TC-AUTH-07: Главная без авторизации *(обновлён)*
**Предусловия:** cookies очищены.  
**Шаги:** открыть `http://localhost:5173/`.  
**Ожидание:** **нет** редиректа на `/login`; таблица с данными; `header-login-btn`; нет `add-bike-btn`.

### TC-AUTH-08: GET /bikes без cookie *(актуализирован — совпадает с v2.1)*
**Шаги:** GET `http://localhost:5000/api/bikes` без cookie.  
**Ожидание:** **200** и список байков (публичный просмотр).

### TC-AUTH-09: Возврат с login на главную *(новый)*
**Шаги:** `/login` → `back-to-home-btn`.  
**Ожидание:** переход на `/`, таблица без входа.

## 2. Роли и доступ

### TC-ROLE-01: Аноним не видит CRUD *(обновлён, был guest)*
**Предусловия:** не авторизован.  
**Шаги:** открыть `/`.  
**Ожидание:** нет `add-bike-btn`, `edit-bike-*`, `delete-bike-*`; есть `actions-readonly-placeholder`; фильтры и сортировка работают.

### TC-ROLE-02: Mechanic — add + edit, без delete
**Предусловия:** вход mechanic.  
**Ожидание:** `add-bike-btn` и `edit-bike-*` есть; `delete-bike-*` нет; edit открывает модалку.

### TC-ROLE-03: Admin — полный доступ
**Предусловия:** вход admin.  
**Ожидание:** add, edit, delete на месте.

### TC-ROLE-04: Mechanic DELETE через API → 403
**Шаги:** DELETE `/api/bikes/{id}` с cookie mechanic.  
**Ожидание:** 403; байк в базе.

### ~~TC-ROLE-05~~ — **УДАЛЁН** (guest POST)

### TC-ROLE-05: POST без авторизации → 401 *(новый, номер переиспользован)*
**Шаги:** POST `/api/bikes` с валидным телом, без cookie.  
**Ожидание:** 401; байк не создан.

### TC-ROLE-06: Admin DELETE через API → 204
**Шаги:** DELETE с cookie admin.  
**Ожидание:** 204; байк удалён из списка.

## 3. VIN при редактировании *(новый блок)*

### TC-BIKE-EDIT-VIN-01: Успешная смена VIN
**Предусловия:** admin или mechanic; байк с уникальным VIN.  
**Шаги:** edit → изменить VIN на новый уникальный 17-символьный → сохранить.  
**Ожидание:** модалка закрыта; в таблице новый VIN; `edit-bike-{новый_vin}`.

### TC-BIKE-EDIT-VIN-02: Дубликат VIN при редактировании
**Шаги:** edit → ввести VIN другого существующего байка → сохранить.  
**Ожидание:** `form-server-error`: «Мотоцикл с таким VIN уже существует в базе»; запись не изменена.

### TC-BIKE-CREATE-VIN-01: Дубликат VIN при создании
**Шаги:** создать байк с VIN, который уже есть в таблице.  
**Ожидание:** то же сообщение об ошибке.
